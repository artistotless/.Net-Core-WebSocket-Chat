Chat Websocket API 1.0.0
  • artistotless

Публичный API веб-сокетного чат проекта.

Servers

  • wss://localhost:5001/chatwssadmin

    Endpoint для подключения операторов. Сведения о статусе авторизации оператора WebSocket сервер берет из HttpContext. Перед подключением, необходимо авторизоваться в админ панеле. https://localhost:5001/admin/login

  • wss//localhost:5001/chatwssplayer

    Endpoint для подключения игроков. Требуется token, который можно получить POST запросом подробнее

    token
    required
    string

    Вставьте токен в url подключения к вебсокету - wss//localhost:5001/chat?token={token}

Operations

  • PUB sendMessage

    Отправить сообщение в чат

    Available only on servers:

    • admin
    • player

    Accepts one of the following messages:

    • #0operatorMessage

      Сообщение от оператора к определенному игроку

      object allOf
      uid: operatorMessage
      restricted any
      playerId
      required
      integer

      ID игрока, которому нужно отправить сообщение.

      message
      required
      string

      Текстовое сообщение

      object
      uid: default

      Этот объект всегда является частью полезной нагрузки.

      type
      required
      string

      Тип запроса, например - sendMessage, setMessageRead и т.д

      Additional properties are allowed.

      Additional properties are allowed.

      Examples

      • #1 Example - sendMessage

        Пример отправки сообщения игроку.

        {
  "type": "sendMessage",
  "data": {
    "playerId": 3,
    "message": "Пожалуйста ожидайте, я попробую вам помочь.."
  }
}
    • #1playerMessage

      Сообщение от игрока к операторам

      string

      Ваше обращение в техподдержку

      object
      uid: default

      Этот объект всегда является частью полезной нагрузки.

      type

    required

    Тип запроса, например - sendMessage, setMessageRead и т.д

  }
}

  • PUB getMessages

    Запросить список сообщений.

    Available only on servers:

    • admin

    Accepts the following message:

    getMessages

    Получение списка сообщений

    object allOf

    По умолчанию (page=1) сервер возвращает 0 - N последних сообщений. При page=2, сервер вернет следующую порцию сообщений N - N*2. Это необходимо для реализации пагинации.

    restricted any
    page
    integer
    playerId
    integer
    object
    uid: default

    Этот объект всегда является частью полезной нагрузки.

    type
    required
    string

    Тип запроса, например - sendMessage, setMessageRead и т.д

    Additional properties are allowed.

    Additional properties are allowed.

    Examples

    • #1 Example - Пример запроса 3 страницы сообщений в беседе с игроком playerId = 10
      {
  "type": "getMessages",
  "data": {
    "page": 3,
    "playerId": 10
  }
}
    • #2 Example - Пример запроса 1 страницы сообщений от всех игроков
      {
  "type": "getMessages",
  "data": {
    "page": 1
  }
}
    • #3 Example - Пример запроса всех без исключения сообщений
      {
  "type": "getMessages",
  "data": {}
}

    restricted any

    required
    string

    Additional properties are allowed.

    required
    object allOf
    uid: player
    restricted any
    id
    required
    integer
    name
    required
    string

    Additional properties are allowed.

    object
    uid: default

    Этот объект всегда является частью полезной нагрузки.

    type
    required
    string

    Тип запроса, например - sendMessage, setMessageRead и т.д

    Additional properties are allowed.

    Additional properties are allowed.

    Examples

    • #1 Example - Пример полученных сообщений.

      В messages хранится массив данных типа `message`. Count содержит информацию о кол-ве сообщений по запросу - эта информация необходима для реализации пагинации. 

      {
  "type": "getMessages",
  "data": {
    "count": 0,
    "messages": [
      {
        "messageId": 36,
        "text": "hello",
        "timeStamp": "10.01.2022 10:40",
        "operator": {
          "name": "James"
        },
        "player": {
          "id": 1,
          "name": "Dean"
        }
      }
    ]
  }
}
